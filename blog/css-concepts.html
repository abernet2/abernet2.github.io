CSS Concepts Reflection

11.6.2015

What is the difference between margin, border, and padding?

Padding

Spacing that occurs outside of your content. If you change the element's background, that will affect the padding area.

Border

The dividing line between padding space and margin. Borders are optional, they can have many different colors and styles.

Margin

Spacing that goes outside of your border. Will not be affected if you change the element's background. Can also have negative values for positioning purposes.

What are the best practices associated with using classes vs. ids?

The basic difference is that classes can occur multiple times, and ids can only occur once. There are several implications of this. IDs basically are used to identify unique objects so you can grab and edit something that is uniqe. CSS Tricks suggests thinking of classes as barcodes, and IDs as serial numbers.

Classes, on the other hand, are used to condense code so you don't have to repeat yourself. Objects can have multiple classes, classes can also be applied to multiple elements at the same time. You could, for instance, make an invisible class and add/subtract it from elements using Javascript to hide that object.

Bottome line, in CSS the difference between the two is entirely semantic (use '.' or '#'). Once we get to Javascript, the functionality really changes, and if you reuse and id the code will not pass an HTML validator, which is important.

What are the differences between relative, absolute, fixed, and static positioning?

Static

HTML elements by defaul have the static positioning element. The static feature places all objects flat on the page relative to other objects. This gives a flow to html elements so that two never, by default, occupy the same position.

Relative

Relative positioning, confusingly, places objects relative to their defaul static positioning. This positioning allows you to treat the static position as a starting point and to use the left, right, top, bottom CSS to position your element around that starting point.

Absolute

Absolute positioning allows you to hard-wire the position of your element. Like the relative positioning, you can use left, right, etc to move your element, only the starting point is the parent element. When you use bottom with relative positioning, you move from the relative bottom, but when you use the bottom attribute in absolute, you move relative to the bottom of the parent element.

Absolute gets this effect by taking your element out of the flow that we spoke of with static positioning. If you think of the page as a grid, absolute instead places your element at the (0,0) position of the first parent element that is not Relatively positioned or Absolutely positioned. In english: it makes it possible to place objects at the top or bottom of divs (or other elements) very easily.

Fixed

Maybe the easiest to understand, fixed positioning fixes an element relative to the screen, not the html page. Have you ever been to a website where the nav-bar stays in place no matter how far you scroll? Or a site that has a sidebar that doesn't react to scrolling? Those are fixed elements.

What does it mean to display inline vs using inline block?

When you use the inline display method, you tell the browser to get as much on one line as it can. Usually this means that it will put text and links on a line so that you can read from left to right and so it can adjust the words per line.

A drawback of the inline display method, is that you generally can't set height or width of that element. The element's size is always relative to its content. One way to fix this is to use display as a block. BUT, doing so will put your element on its own line.

Inline block utilizes the blockiness of block (setting you own size), and still positions your element in-line with text or other inline elements.
